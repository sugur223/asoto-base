# あそとステップ管理 - 設計ドキュメント

## エグゼクティブサマリー

「誰もがあそぶように、シゴトができる社会」を実現するための成長支援システム「あそとステップ管理」の設計書です。

**核心的な設計思想**
- **統一されたフレームワーク** - 9ステップという共通言語でコミュニティをつなぐ
- **柔軟な成長経路** - 個人の状況・性格・ペースに合わせてカスタマイズ
- **データドリブン進化** - ユーザーの行動から学習し、最適な道筋を提案

**3つの価値提供**
1. **可視化** - 見えない成長を見える化する
2. **ガイド** - 迷わず進める道標を提供する
3. **つながり** - 一人じゃない、仲間がいる

## 設計哲学：標準化とカスタマイズのバランス

### 根本的な問い

**「ロードマップは統一すべきか、個々人でカスタマイズすべきか？」**

この問いに対する答えは、**どちらでもなく、その両方を取る**です。

### 3つのアプローチの比較

| アプローチ | メリット | デメリット | 適用場面 |
|-----------|---------|-----------|---------|
| **完全統一** | ・分かりやすい<br>・共通言語になる<br>・シンプル | ・個人差を無視<br>・押し付け感<br>・合わない人が離脱 | 初心者、明確な答えが欲しい人 |
| **完全カスタマイズ** | ・個人最適化<br>・自分のペース<br>・多様性尊重 | ・選択肢過多で迷う<br>・共通言語がない<br>・複雑 | 上級者、自律性が高い人 |
| **ハイブリッド** | ・両方の良いとこ取り<br>・段階的に自由度UP<br>・データで最適化 | ・設計が複雑<br>・説明が必要 | **推奨アプローチ** |

### 採用する設計：アダプティブロードマップ

**コンセプト**
「統一されたフレームワーク + 柔軟な成長経路」

```
【固定要素】
├─ 9つのステップ（マイルストーン）
├─ 3つの成長要素（測定軸）
└─ 達成の本質的条件

【柔軟要素】
├─ ステップへの到達方法（複数ルート）
├─ クエストの選択（必須 vs 選択可）
├─ ペース配分（スピード vs じっくり）
└─ 支援の形（AI vs 人 vs 両方）
```

**なぜこのアプローチか**

1. **コミュニティの共通言語が必要**
   - 「私は今ステップ5だよ」という会話ができる
   - 先輩が「ステップ4の時はこうだった」とアドバイスできる
   - 進捗を比較・共有できる

2. **でも個人差は無視できない**
   - 「遊び発」vs「仕事発」の2ルート
   - 内向型 vs 外向型
   - 時間豊富 vs 超多忙
   - じっくり派 vs スピード派

3. **データから学習して進化できる**
   - 「このタイプの人にはこのクエストが効く」
   - 躓きポイントの早期発見
   - レコメンドの精度向上

## 全体構造

### ビジョン

**「誰もがあそぶように、シゴトができる社会を。」**

仕事と遊びの境界を溶かし、人生の大半を占める「シゴト」を子ども時代の純粋な「遊び」のように心から楽しみながら、人や社会に貢献できる状態を目指す。

### 3つのフェーズ × 9つのステップ

```
🟡 フェーズ1: あそび期間（準備期） - ステップ1-3
   テーマ: 豊かな人間関係を築く
   ゴール: 信頼できる仲間とのつながり
   
🔴 フェーズ2: あそと構想期（変容前期） - ステップ4-6
   テーマ: 遊びが仕事になる基盤をつくる
   ゴール: 価値創出と対価獲得の体験
   
🔴 フェーズ3: あそと成長期（変容後期） - ステップ7-9
   テーマ: 人間関係は仕事の選び、ずっと遊びが仕事になる
   ゴール: 自律的なキャリアと互恵共栄
```

### 成長曲線

```
働く力の質量 ↑
              |                                あそと・ライフワーク
              |                              ／
              |                            ／
              |                          ／
              |        あそと構想期    ／
              |          ／＼        ／
              |        ／    ＼    ／
              |  あそび期間    ＼／
              |  ／￣￣￣￣
              |／
              +--------------------------------→ かかる時間
             仕事・ライフワーク
```

### 2つの成長ルート

#### ルート1: あそび発（遊び → 仕事）

**特徴**
- 会社/家庭の外に出る
- 興味・好奇心から始める
- 徐々に価値創出・対価獲得へ

**プロセス例**
```
趣味の活動
  ↓
コミュニティで企画運営
  ↓
スキルシェア・ワークショップ
  ↓
副業・プロジェクト受注
  ↓
起業・独立
```

**向いている人**
- 仕事にストレスを感じている
- 好きなことがある
- 新しいつながりが欲しい
- まずは遊びから始めたい

#### ルート2: しごと発（仕事 → 遊び）

**特徴**
- 既存の仕事を持ちながら
- マインドセット・スキルを変える
- 仕事を遊びのように楽しむ

**プロセス例**
```
今の仕事の見直し
  ↓
社外コミュニティで視野拡大
  ↓
会社での挑戦（新規プロジェクト等）
  ↓
転職・社内異動
  ↓
複業・パラレルキャリア
```

**向いている人**
- 今の仕事にやりがいを感じたい
- 転職を考えている
- スキルアップしたい
- 仕事の意味を再定義したい

#### ルート3: ハイブリッド（両方同時）

**特徴**
- 仕事も遊びも両方変える
- 相乗効果を狙う
- 最も変容が大きい

**向いている人**
- 時間とエネルギーに余裕がある
- 大きな変化を求めている
- 柔軟に両立できる

## 3つの成長要素

### あそとで磨く3要素 vs 会社で磨く3要素

| あそとで磨く | 説明 | 会社で磨く | 説明 |
|------------|------|-----------|------|
| **① 関係性**<br>Relationality | 偶発的・共感的・横のつながり<br>人との出会いが遊びの起点 | **① 組織性**<br>Organization | 明確な役割・上下関係・縦の構造<br>安定した組織運営 |
| **② 多動性**<br>Multimobility | まずやってみる・探索的<br>行動量が偶発を引き寄せる | **② 一貫性**<br>Consistency | 計画的・遂行的・安定志向<br>目標達成と効率性 |
| **③ 感受性**<br>Sensitivity | 感じて動く・直感的<br>自分の「ねくねく」を感じ取る | **③ 論理性**<br>Logic | 考えて判断・理性的<br>合理的な意思決定 |

**補完関係による「仕事で遊ぶ」状態の実現**

「会社で磨く理性」と「あそとで育む感性」の両輪がそろうことで、仕事はもっと面白くなる。

### 測定とポイント制度

各要素を0-200ptで可視化し、レーダーチャート表示

```
関係性（Relationality）: 0 → 200pt
├─ 場への参加回数（イベント、グループ）
├─ 対話の深さ（内省ログ、深い会話）
├─ つながった人数（新規つながり）
├─ 信頼関係の強さ（相互フィードバック）
└─ 紹介・マッチング数（他者を繋げた数）

多動性（Multimobility）: 0 → 200pt
├─ イベント参加数（継続性）
├─ 新規挑戦数（初めての活動）
├─ 企画実行数（自主企画）
├─ 継続日数（ストリーク）
└─ 行動の幅（多様な活動）

感受性（Sensitivity）: 0 → 200pt
├─ 内省ログ数（振り返りの頻度）
├─ 気づきの言語化（質の高い内省）
├─ 自己理解の深化（診断完了、強み発見）
├─ フィードバック受容（他者の声を受け取る）
└─ 感情の表現（喜び、悩み、願い）
```

**バランスの重要性**

どれか1つが高ければいいわけではなく、3要素のバランスが重要。

```
【理想的なバランス】（ステップ9到達時）
   関係性
      ▲
     /  \
180pt   180pt
   /  ●  \
  /        \
 /____●____\
多動性 180pt 感受性
```

## アダプティブロードマップ設計

### レイヤー構造

```
【レイヤー1】コアステップ（統一・必須）
├─ 9つの基本ステップは全員共通
├─ 大きなマイルストーンとして機能
├─ コミュニティの共通言語
└─ 本質的な達成条件は変わらない

【レイヤー2】ルート選択（パターン化されたカスタマイズ）
├─ ルートA「あそび発」（遊び→仕事）
├─ ルートB「しごと発」（仕事→遊び）
├─ ルートC「ハイブリッド」（両方同時）
├─ 初回診断で推奨ルートを提示
└─ 途中でルート変更も可能

【レイヤー3】クエストカスタマイズ（個別最適化）
├─ 性格タイプ別クエスト（MBTI、エニアグラム等）
├─ 興味・強み別クエスト（VIA診断結果等）
├─ 時間制約別クエスト（多忙度に応じて）
├─ ペース別クエスト（じっくり vs スピード）
└─ AIが毎週「あなたにおすすめ」を提案

【レイヤー4】リアルタイム適応（AI駆動）
├─ 行動データから学習
├─ 躓きポイントの早期検知
├─ 動的なクエスト再提案
└─ モチベーション管理
```

### 初回オンボーディング：ルート診断

新規ユーザーは最初に診断を受けて、推奨ルートを決定

**診断項目**
```
1. 現在の状況
   □ 仕事に満足している / していない
   □ 休日の過ごし方（消費的 / 創造的）
   □ 社外のつながり（多い / 少ない）
   
2. 志向性
   □ 新しいことを始めたい / 今を改善したい
   □ 遊びが好き / 仕事が好き / どちらも
   □ 独立志向 / 組織内でのキャリアアップ志向
   
3. リソース
   □ 時間の余裕（週に使える時間）
   □ エネルギーレベル（高い / 普通 / 低い）
   □ 経済的余裕（投資できる金額）
   
4. 性格・強み
   □ 内向型 / 外向型
   □ 計画型 / 柔軟型
   □ 独立志向 / 協調志向
```

**診断結果の例**
```
┌─────────────────────────────────┐
│ あなたにおすすめのルート           │
├─────────────────────────────────┤
│ 🎮 ルートA「あそび発」             │
│                                   │
│ あなたは...                       │
│ ・仕事にやや不満を感じている       │
│ ・新しいつながりを求めている       │
│ ・好きなことから始めたいタイプ     │
│                                   │
│ おすすめの最初の一歩：             │
│ 1. 興味のあるイベントに参加        │
│ 2. 趣味仲間を見つける              │
│ 3. 小さな企画を立ち上げる          │
│                                   │
│ [このルートで始める]               │
│ [他のルートも見る]                 │
└─────────────────────────────────┘
```

### モード選択：ガイドの強度

ユーザーは自分に合った「ガイドの強度」を選べる

**1. ガイドモード（初心者・明確な道筋が欲しい人向け）**
```
特徴:
├─ システムが「これをやろう」と1つの道筋を提示
├─ 迷わない、考えすぎない
├─ ステップバイステップで進む
└─ 達成感を得やすい

推奨ユーザー:
├─ 初めての人
├─ 決断疲れしやすい人
├─ 明確な指示が欲しい人
└─ まずは成功体験が欲しい人
```

**2. エクスプローラーモード（自由派・自己決定したい人向け）**
```
特徴:
├─ 複数のクエストから自分で選ぶ
├─ 飛び級OK、順番変更OK
├─ 自分のペースで探索
└─ 自己決定感を重視

推奨ユーザー:
├─ 自律性が高い人
├─ 選択肢が多い方が好き
├─ 自分で道を切り開きたい人
└─ 既にコミュニティに慣れている人
```

**3. ブレンドモード（推奨・バランス型）**
```
特徴:
├─ AIがパーソナライズして提案
├─ 基本はガイドに従う
├─ でも「今週はこっちがやりたい」も選べる
├─ 柔軟に調整可能
└─ データから学習して最適化

推奨ユーザー:
├─ 迷ったらこれ（デフォルト）
├─ ガイドも欲しいし自由も欲しい
├─ 状況に応じて変えたい
└─ AIの提案を試したい
```

### ルート別クエスト設計（ステップ4の例）

**【ステップ4：価値を創る】**

**コア達成条件（全ルート共通）**
- ✅ 自主企画を1つ実行
- ✅ 自己理解ツール完了
- ✅ できること/やりたいことリスト作成
- ✅ 多動性ポイント50pt以上

---

**ルートA「あそび発」向けクエスト**

*メインクエスト*
```
🎨 「好きなこと」から小さな企画を作る
├─ 達成条件: 自分の趣味・興味から1つ企画実行
├─ 例: 読書会、もくもく会、フィールドワーク
├─ 所要時間: 2-4時間（企画+実施）
└─ 報酬: 多動性+25pt、創造者バッジ
```

*サブクエスト（選択可）*
```
A1: 「楽しい！」を価値に変える練習
    └─ 自分が楽しんだことを、他者にも共有する

A2: 遊び仲間を3人巻き込む
    └─ 一人じゃなく、仲間と一緒に何かを作る

A3: 「これ面白そう」アンテナを立てる
    └─ 日常で気になったことをメモ、企画の種に
```

---

**ルートB「しごと発」向けクエスト**

*メインクエスト*
```
💼 今の仕事の中で「これは価値だ」を見つける
├─ 達成条件: 現在の仕事から1つ価値を抽出・言語化
├─ 例: 得意な業務を社外で教える、スキルシェア
├─ 所要時間: 2-4時間（棚卸し+実践）
└─ 報酬: 多動性+25pt、創造者バッジ
```

*サブクエスト（選択可）*
```
B1: 会社で磨いたスキルを外で試す
    └─ 社内の専門性を社外コミュニティで発揮

B2: 「やらされ仕事」を「やりたい仕事」に転換
    └─ 同じ業務でも視点を変えると価値が見える

B3: 社内で新しい挑戦をする
    └─ 手を挙げていなかったプロジェクトに参加
```

**性格タイプ別クエスト（AIがマッチング）**

*内向型向け*
```
🤫 1対1での価値提供から始める
├─ 大勢の前は苦手→まずは1人の困りごとを解決
├─ 例: メンタリング、相談に乗る、個別レクチャー
└─ 報酬: 感受性+15pt、関係性+10pt
```

*外向型向け*
```
🎉 イベント企画で人を集める
├─ 人と会うのが好き→その強みを活かす
├─ 例: 交流会、ワークショップ、オフ会
└─ 報酬: 関係性+15pt、多動性+10pt
```

*思考型向け*
```
🧠 ワークショップ設計で体系化
├─ 論理的に考えるのが得意→構造化された価値提供
├─ 例: ステップバイステップの学習プログラム
└─ 報酬: 多動性+20pt、感受性+10pt
```

*感情型向け*
```
💖 共感の場づくりでつながる
├─ 人の気持ちに寄り添える→安全な場を作る
├─ 例: 対話の場、悩み相談会、シェアリング
└─ 報酬: 関係性+20pt、感受性+10pt
```

---

**時間制約別クエスト**

*多忙な人向け（週1-2時間）*
```
⏱️ 30分でできる価値提供
├─ 短時間で完結するクエスト
├─ 例: 質問箱、スキルシェア動画1本、1on1 30分
└─ 低負荷でも確実に進める設計
```

*時間に余裕がある人向け（週5時間以上）*
```
🕐 じっくり準備して大きな企画
├─ 時間をかけて質の高いものを作る
├─ 例: 連続講座、大型イベント、プロジェクト立ち上げ
└─ 大きな達成感と影響力
```

**過去の行動パターン別（AIが自動判定・提案）**

```
パターン分析:
├─ よく参加するイベントタイプ → 類似の企画を推奨
├─ 内省ログのテーマ → 興味領域を抽出して企画案
├─ つながった人のニーズ → 「〇〇さんが〇〇で困ってる」
└─ 過去の成功体験 → 「前回〇〇で輝いてたよね」

提案例:
「〇〇さんへのおすすめクエスト」
└─ 「デザイン思考ワークショップ」
    理由: あなたは過去に創造的な活動で高評価
         3人があなたのデザインスキルを求めています
```

### クエストライブラリの構造

全てのクエストをライブラリ化し、タグ付け

**クエストのメタデータ**
```
・タイトル・説明
・対象ステップ（1-9）
・ルート（A/B/C/共通）
・難易度（★～★★★★★）
・所要時間（分）
・性格マッチ（内向/外向/思考/感情等）
・タグ（#企画 #交流 #知的好奇心 等）
・報酬（XP、ポイント、バッジ）
・成功率（全ユーザー平均）
・躓きポイントと対策
```

### AI推薦エンジンの仕組み

**入力データ**
```
ユーザー属性:
├─ 診断結果（性格、興味、強み）
├─ 選択したルート
├─ 選択したモード
└─ 時間制約

行動履歴:
├─ 過去のクエスト達成状況
├─ イベント参加履歴
├─ 内省ログのテーマ
├─ つながった人の特徴
└─ 3要素ポイントのバランス

コンテキスト:
├─ 現在のステップ
├─ 直近の躓きポイント
├─ コミュニティのニーズ
└─ 季節・時期（年度末、夏休み等）
```

**推薦アルゴリズム**

## 躓きポイントとサポート策

### 全ステップ横断の共通躓きポイント

#### カテゴリ1: 時間・エネルギー不足（全ステップ）

**症状**
- 仕事が忙しくて参加できない
- 優先順位が下がる
- 継続できない

**サポート策**
```
⏰ マイクロコミットメント
├─ 5分でもOKな参加形式
├─ 「今週は無理せず見守るだけ」も選択肢
└─ 低負荷な関わり方の提示

🎲 習慣化設計
├─ 毎週同じ時間・曜日の固定
├─ 「ついで」でできる設計
└─ 生活リズムへの組み込み
```

#### カテゴリ2: 心理的障壁（ステップ1-3で顕著）

**症状**
- 不安、恐怖、緊張
- 完璧主義で動けない
- 他者の目が気になる

**サポート策**
```
🛡️ 心理的安全性の確保
├─ 失敗OK文化の明示
├─ 匿名・限定公開の選択肢
└─ ジャッジしない場の設計

🎮 プレイフル設計
├─ 「実験」「お試し」のフレーミング
├─ ゲーム要素で軽やかに
└─ 小さな成功体験の積み重ね
```

## まとめ

### あそとステップ管理の本質

このシステムは単なるタスク管理ツールではなく、**「人生の変容を支援する成長プラットフォーム」**です。

### 3つの核心価値

**1. 可視化**
- 見えない成長を見える化する
- 小さな進歩も積み重ねも全て記録

**2. ガイド**
- 迷わず進める道標を提供する
- 躓いた時のサポートが自動で届く
- でも、押し付けない、選択肢を与える

**3. つながり**
- 一人じゃない、仲間がいる
- 同じ道を歩く仲間との出会い
- 互恵共栄のエコシステム

### 成功の鍵

**技術 × 人間理解**
- 技術だけでは人は変わらない
- 人間心理の深い理解が必要

**統一 × カスタマイズ**
- 共通言語としての9ステップ
- でも一人ひとりに合わせた道筋

**遊び心 × 本気**
- ゲーム要素で楽しく続ける
- でも目指すのは本物の変容

**個人 × コミュニティ**
- 一人ひとりの成長
- でも支え合う仲間がいる

## よくある質問（FAQ）

**Q1. 9ステップを飛ばすことはできますか？**

A. エクスプローラーモードを選択すれば柔軟に進められます。ただし、ステップには依存関係があるため、飛ばしすぎると後で困る可能性があります。

**Q2. 必ず9ステップ全て達成しないとダメですか？**

A. いいえ。9ステップは「目指せる目標」であり、強制ではありません。大切なのは「あなたがどうありたいか」です。

**Q3. ルートを途中で変更できますか？**

A. はい、いつでも変更可能です。柔軟に調整してください。

**Q4. AIの推薦を無視してもいいですか？**

A. もちろんです。AIはあくまで「提案」であり、最終的に決めるのはあなたです。

**Q5. 他の人とステップ進捗を比較されるのは嫌です。**

A. 進捗は基本的にプライベートで、本人が公開を選択しない限り他者には見えません。

**Q6. 途中でやめてもいいですか？**

A. もちろんです。いつでも戻ってこれます。asotobaseはあなたの居場所です。

**Q7. このシステムで本当に人生が変わりますか？**

A. システムはあくまで「道具」です。変わるのはあなた自身の行動と選択によってです。ただ、このシステムはその変容を「支援」し、「加速」させることができます。

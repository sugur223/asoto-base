# 内省ログ機能 - 詳細設計

## 1. 機能概要

ユーザーが日々の活動を振り返り、学びや気づきを記録するための個人専用スペース。やりっぱなしを防ぎ、深い内省を通じて成長を促進する。

## 2. ユーザーストーリー

### 初心者ユーザー（山田さん）のケース

**背景**
- 行動はするが振り返りの習慣がない
- 何を書けばいいかわからない

**ストーリー**
1. イベント参加後、システムから「振り返りログを書きませんか？」という通知が届く
2. 「書いてみる」をクリック
3. テンプレートが表示される：
   - 「今日やったこと」
   - 「どんな発見や学びがありましたか？」
   - 「次に活かせそうなことは？」
4. 空欄に沿って書くだけで、自然と振り返りができる
5. 投稿後、AIから「その発見、素晴らしいですね！」とフィードバックが届く
6. 5ポイント獲得

### アクティブユーザー（鈴木さん）のケース

**背景**
- すでに振り返りの習慣がある
- より深い内省がしたい

**ストーリー**
1. 週末に1週間の活動を振り返る
2. フリーフォームで長文のログを記録
3. タグ付け：#読書会 #気づき #コミュニケーション
4. 過去のログを検索：「コミュニケーション」タグで絞り込み
5. 3ヶ月前の自分と今の自分を比較
6. 「コミュニケーションスキルが確実に向上している」と実感
7. AIコーチングに「どうすればもっと成長できますか？」と質問
8. AIが過去のパターンから具体的な提案をくれる

### コミュニティリーダー（佐藤さん）のケース

**背景**
- イベントを主催している
- 参加者の声を知りたい

**ストーリー**
1. イベント後、参加者の公開ログを確認
2. 「今日の読書会、とても刺激的でした」という投稿を発見
3. コメントで「ありがとうございます！次回も楽しみにしています」と返信
4. 参加者のフィードバックを次回のイベント企画に活かす

## 4. データモデル

### Log（内省ログ）

```typescript
interface Log {
  id: string;
  userId: string;
  title: string;
  content: string; // Markdown対応
  tags: string[];
  visibility: 'private' | 'public'; // 非公開/公開
  createdAt: Date;
  updatedAt: Date;

  // 関連データ
  relatedEventId?: string;
  relatedGoalId?: string;
  relatedStepId?: string;

  // メタデータ
  template?: string; // 使用したテンプレート名
  readingTime?: number; // 読むのにかかる時間（分）

  // Relations
  user: User;
  comments: LogComment[];
```

## 5. ユースケース詳細

5. ユーザーが各項目を入力

8. ユーザーが公開設定を選択
9. ユーザーが「AIにフィードバックをもらう」ボタンをクリック（任意）
10. システムがAIフィードバックを生成して表示
11. ユーザーが「投稿する」ボタンをクリック
12. システムがログをデータベースに保存

14. システムがログ詳細画面にリダイレクト
15. システムが「ログを投稿しました！」という通知を表示

**代替フロー**
- 9a. ユーザーが「下書き保存」を選択
  - 9a1. システムがログを下書きとして保存
  - 9a2. ログ一覧に「下書き」タグで表示

**事後条件**
- 新しいログがデータベースに保存される
- （公開設定の場合）他のユーザーが閲覧可能になる
- ポイントが付与される

### UC-2: ログを閲覧する（他のユーザー）

**前提条件**
- ユーザーがログイン済み
- 公開ログが存在する

**正常フロー**
1. ユーザーがダッシュボードまたはイベント詳細ページで「参加者のログ」を表示
2. システムが公開ログの一覧を表示
3. ユーザーが気になるログをクリック
4. システムがログ詳細を表示
5. ユーザーがログを読む
6. ユーザーがコメント欄に感想や共感を入力
7. ユーザーが「投稿」ボタンをクリック
8. システムがコメントを保存
9. システムがログ作成者に通知
10. コメント投稿者に3ポイント付与

**事後条件**
- コメントが保存される
- ログ作成者が通知を受け取る

### UC-3: AIフィードバックを受け取る

**前提条件**
- ユーザーがログを作成済み

**正常フロー**
1. ユーザーがログ作成時または作成後に「AIにフィードバックをもらう」をクリック
2. システムがログの内容をAIに送信
3. AIがログを分析：
   - 内容の要約
   - 気づきのポイント抽出
   - 成長の兆候検出
   - 次のアクション提案
4. システムがAIフィードバックを生成
5. システムがフィードバックをログに紐付けて保存
6. システムがフィードバックを表示
7. （パターン検出時）システムが「あなたの関係性スコアが向上しています」などのインサイトを表示

**事後条件**
- AIフィードバックが保存される
- ユーザーが具体的な気づきや提案を得る

### UC-4: 過去のログを検索・振り返る

**前提条件**
- ユーザーがログイン済み
- 過去にログを投稿している

**正常フロー**
1. ユーザーがログ一覧画面で検索バーに「コミュニケーション」と入力
2. システムがタグまたは本文に「コミュニケーション」を含むログをフィルタリング
3. システムが検索結果を表示（時系列順）
4. ユーザーが3ヶ月前のログをクリック
5. システムがログ詳細を表示
6. ユーザーが当時の自分と今の自分を比較
7. ユーザーが「この頃と比べて、確実に成長している」と実感

**代替フロー**
- 2a. タグフィルターを使用
  - 2a1. ユーザーが「タグで絞り込み」から「#コミュニケーション」を選択
  - 2a2. システムが該当するログを表示

**事後条件**
- ユーザーが過去の振り返りを再確認できる
- 成長を実感できる

## 7. AIフィードバックのロジック

### 分析項目

1. **感情分析**
   - ポジティブ/ネガティブ/中立の判定
   - 感情の強度

2. **キーワード抽出**
   - 重要なキーワードやフレーズ
   - 繰り返し出現するテーマ

3. **あそと3要素の分類**
   - 関係性に関する記述
   - 多動性に関する記述
   - 感受性に関する記述

4. **成長の兆候検出**
   - 過去のログとの比較
   - スキルや考え方の変化

5. **アクション抽出**
   - 「〜してみよう」「〜する予定」などの行動意図

### フィードバックタイプ

#### 励まし型（encouragement）

```
素晴らしい気づきですね！○○について深く考えていることが
伝わります。このような振り返りを続けることで、さらに成長
できるはずです。
```

#### 提案型（suggestion）

```
次回は以下のことを試してみてはいかがでしょうか？
• ○○について、さらに深掘りしてみる
• ○○さんに直接フィードバックをもらってみる
• ○○を実践してみて、結果を記録する
```

#### パターン認識型（pattern）

```
過去3ヶ月のログを分析したところ、あなたは「○○」に関する
気づきが多い傾向にあります。これはあなたの強みになりつつ
あるかもしれません。
```

#### 成長可視化型（growth）

```
3ヶ月前のログと比較すると、「コミュニケーション」に関する
記述が増え、より具体的になっています。確実に成長している
証拠です📈
```

## 10. プライバシー・セキュリティ

### 公開設定の仕組み

- **非公開（private）**: 本人のみ閲覧可能
- **公開（public）**: コミュニティメンバー全員が閲覧可能

### データ保護

- ログデータは暗号化して保存
- 削除したログは完全に削除（論理削除ではなく物理削除も選択可能）
- エクスポート機能：自分のログをJSON/Markdown形式でダウンロード
